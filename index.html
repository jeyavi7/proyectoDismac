Planteamiento del Problema

Árbol de Problema

Efecto principal: Pérdidas económicas y clientes insatisfechos
Problema central: Ineficiencia en la gestión de ventas e inventarios
Causas: Procesos manuales, falta de visibilidad en tiempo real, sistema legado obsoleto.

Análisis Estructurado

Modelo Ambiental

Declaración de Propósitos: El sistema debe gestionar ventas, inventarios, clientes y generar reportes. Debe integrarse con sistemas externos de proveedores y pasarelas de pago.
📊 Diagrama de Contexto
[Entidades externas: Cliente, Proveedor, Administrador] → Sistema de Ventas DISMAC

Modelo de Comportamiento

DFD por Niveles: Nivel 0: Procesos principales (Ventas, Inventario, Reportes). Nivel 1: detalle de validación de pago, actualización de stock, etc.
DER (Diagrama Entidad-Relación): Principales entidades: Cliente, Producto, Venta, DetalleVenta, Proveedor, Usuario. Relaciones: Venta pertenece a Cliente, DetalleVenta asocia Venta y Producto.

Orientado a Objetos

Diagrama de Clases

Clases principales: Venta, Producto, Cliente, Inventario, Proveedor. Atributos y métodos definidos con visibilidad privada/pública.

Casos de Uso

Actores: Cajero, Administrador, Proveedor. Casos: Registrar venta, gestionar producto, consultar stock, emitir reporte.

Diagrama de Secuencia

Interacción para "Realizar compra": Cliente -> Cajero -> Sistema -> BaseDatos -> Confirmación.
